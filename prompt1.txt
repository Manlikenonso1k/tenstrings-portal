If a duplicate is found, it should be logged as "Skipped" rather than throwing an error or creating a double entry.

Summary Report: After the import finishes, display a summary notification or modal showing:

Total rows processed.

Number of successful imports.

Number of skipped/duplicate rows.

Validation: Ensure the importer validates required fields (Name, Course, Start Date) and provides a CSV of "failed rows" that the user can download and fix.

OUTPUT: Provide the Importer class logic, the StudentResource action configuration, and instructions for setting up the Laravel Queue Worker to handle the progress bar.
