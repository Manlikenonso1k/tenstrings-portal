Adjust Sidebar Width: On mobile devices, set the sidebar width to exactly 60% of the screen width (w-[60vw]). The goal is to ensure the page content remains partially visible behind the menu overlay.

Overlay Logic: Ensure clicking the darkened overlay (the area outside the 60% menu) also closes the sidebar.

TASK 2: THEME & COLOR SYNCHRONIZATION

Color Matching: The Admin portal styling is the gold standard. Copy the Blue Primary Color configuration from the AdminPanelProvider to the StudentPanelProvider.

Hover & Active States: >    - Force all sidebar menu items and buttons in the Student Portal to use the same Blue color for Hover and Active states as the Admin portal.

Ensure the text contrast is sharp (e.g., white text on blue background) to avoid any dark mode visibility issues.

TASK 3: CSS OVERRIDES (IF NECESSARY)

If Filament's default configuration doesn't allow for the 60% width, inject a custom FilamentView::renderHook or a css file to target .fi-sidebar on mobile screens (@media (max-width: 768px)).

Implementation Guide: Syncing the Blue Theme
To make sure the "Admin Blue" carries over to the Student side immediately, your agent should update the StudentPanelProvider.php like this:

PHP
use Filament\Support\Colors\Color;

public function panel(Panel $panel): Panel
{
    return $panel
        ->id('student')
        ->path('portal')
        ->colors([
            'primary' => Color::Blue, // This syncs the Admin Blue
        ])
        // ... other config
}
